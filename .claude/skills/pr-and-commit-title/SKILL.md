Pull request titles and commit messages should be in the following format:
`[EMOJI] [KEYWORD]: [Description]`

## Formatting Rules

- Start with an emoji that represents the type of change being made.
- Then a space followed by a keyword in uppercase that describes the type of change.
- Then a colon followed by a space and a short description of the change in sentence case.
- Do not end the title or commit message with a period or any punctuation.

## Examples:

- 🔒 SECURITY: Add several domains to the Content Security Policy
- 💄 UI: Update product details section on the product page
- 👽 API: Implement new endpoint for fetching blog posts
- ✨ FEATURE: Add live chat button
- 🔥 REMOVE: Delete unused components
- 📦 PACKAGE: Add Zustand as a new dependency
- 🍱 ASSETS: Add new watch product images
- ♻️ REFACTOR: Update the product details component to use hooks
- 👷 CI: Create new type check workflow file
- 👔 LOGIC: Implement cart state management
- ⚡️ PERFORMANCE: Improve home page load time by lazy loading images
- 📝 DOCS: Update README file with relevant project setup details
- 📈 ANALYTICS: Add track purchase Google Analytics event
- 🔍 SEO: Update meta descriptions on all pages
- 🧪 TESTS: Add end-to-end tests for log in flow

You can optionally use the emojis from Gitmoji (https://gitmoji.dev) as a guide for deciding on a relevant emoji and keyword, if necessary.
